# 123 — Requests & Testing Center

> **1-2-3:** see something → say something → ship something.
> This is where you ask RAND0M for things, report what's broken, and help test
> what's next.

RAND0M is built in the open and shaped by the people who use it. **123** is the
front door for requests, bug reports, feature ideas, and test sign-ups. You don't
need to be an engineer — clear words are enough.

---

## What you can file

| Type | Use it when… | Start with |
| --- | --- | --- |
| 🐛 **Bug** | something is wrong, broken, or confusing | the Bug template |
| ✨ **Feature** | you want RAND0M to do something new | the Feature template |
| 🧪 **Test request** | you want to join a test round / try a beta flow | the Test template |
| 🚀 **Release note** | tracking what's going out and when | the Release template |

## The flow (issue → triage → fix → release)

1. **You file it** with a template (clear title, one issue per report).
2. **Triage** — it's labeled, prioritized, and either accepted, merged with a
   similar request, or sent back for more detail.
3. **Build & validate** — accepted work is built and runs through automated checks
   before it can ship.
4. **Staging** — it goes to a test world first so it can be tried live.
5. **Release** — once it passes, it ships to rand0m.ai on a deliberate release,
   and the request is closed with a note pointing at what changed.

## Good-request etiquette

- **One thing per report.** Two bugs = two reports.
- **Show, don't just tell** — a screenshot or screen recording is worth a
  paragraph.
- **Say where** — which page, which globe layer, which device/browser.
- **Search first** — if it's already filed, add a 👍 or a comment instead of a
  duplicate.
